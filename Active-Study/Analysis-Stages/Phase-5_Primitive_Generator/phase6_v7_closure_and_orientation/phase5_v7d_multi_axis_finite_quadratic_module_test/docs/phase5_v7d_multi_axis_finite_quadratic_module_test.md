# Phase 5 v7d: Multi-Axis Finite Quadratic Module Test

## Claim attacked

The Orthad overset dual-lens construction extends from one doubled cyclic carrier to product finite quadratic modules with cross-axis pairings.

## Product carrier

For moduli `D_i`, define

```text
A = Π_i Z/D_i Z
```

The direct-sum pairing is

```text
B(x,y) = Σ_i x_i y_i / D_i
```

The cross-axis extension used in this package is

```text
B_c(x,y) = Σ_i x_i y_i/D_i + Σ_False c_ij (x_i y_j + x_j y_i) / lcm(D_i,D_j)
```

with representative-invariance and nondegeneracy gates.

## Generated transfer

```text
K_A(x,y) = |A|^(-1/2) exp(-2πi B(x,y))
```

## Generated self-twist

```text
G_A(x) = exp(πi B(x,x))
```

## Verdict

Direct-sum product modules are generated by the current Orthad canon. Nonzero cross-axis modules pass when the inter-axis coupling matrix is retained as module data. The primitive-origin frontier is now precise: derive the inter-axis coupling registry from QBL history, or mark it as extra retained product-module data.

## Boundary
