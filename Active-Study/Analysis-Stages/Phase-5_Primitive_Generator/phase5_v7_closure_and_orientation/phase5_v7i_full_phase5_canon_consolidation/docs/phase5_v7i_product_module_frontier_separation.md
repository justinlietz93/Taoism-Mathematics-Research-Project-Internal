# Product-Module Frontier Separation

The product-module frontier is separated into closed/support layers and open layers.

## Supported

```text
1. Direct sums of doubled cyclic axes.
2. Product modules with retained admissible nondegenerate pairwise coupling.
3. Native shared-boundary generation of selected c_ij.
4. Bounded registry/reachability for swept pairwise classes.
5. Three-plus-axis pairwise tensor composition.
```

## Open

```text
1. Complete arbitrary QBL history classification.
2. Concrete mock-theta finite quadratic module matching.
3. Analytic completion beyond the finite carrier skeleton.
4. Local Lean execution.
```

## Rejected in the tested finite quadratic module layer

```text
Independent cubic/triple latch terms as standalone Weil-compatible tensors.
```

They must project to the pairwise bilinear tensor or fail the quadraticity gate.
